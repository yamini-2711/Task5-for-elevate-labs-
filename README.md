# Network Traffic Analysis Using Wireshark

##   Objective
Capture and analyze live network traffic to identify basic protocols and understand packet-level communication.

##  Tools Used
- Wireshark (free network protocol analyzer)

##   Procedure
1. Installed Wireshark and launched the application.
2. Selected the active network interface (Wi-Fi/Ethernet).
3. Started packet capture.
4. Generated traffic by browsing websites and using ping commands.
5. Stopped capture after a minute.
6. Applied filters to analyze protocols (`DNS`, `TCP`, `ICMP`, `HTTP`).
7. Identified multiple protocols in the captured data.
8. Exported the capture as a `.pcap` file.

##   Protocols Identified
- **DNS** – Resolves domain names to IP addresses  
- **TCP** – Provides reliable communication  
- **ICMP** – Used for ping requests and responses  
- **HTTP/HTTPS** – Web traffic communication  

##  Outcome
Gained hands-on experience in packet capturing and improved understanding of network protocols and traffic analysis.

  Deliverables
- Packet capture file (`.pcap`)
- Analysis report
